## v0.4.0

### 新增功能

#### 接口复制

- 侧边栏接口右键菜单新增「复制」选项，一键复制接口及其关联的所有 Mock 规则
- 复制命名规则：`原名-复制[N]`，自动递增编号
- 复制后自动切换到新接口

#### 切换接口时自动加载历史响应

- 切换接口时自动从历史记录加载最近一次的响应数据，无需重新发送请求即可查看上次结果

### 优化改进

- KeyValueTable 空行初始化逻辑优化，修复外部数据同步后可能缺少空行的问题
- 移除 curl 复制命令中的代理参数（`-x`），避免泄露本地代理配置
- `loadFromEndpoint` 改为异步方法，确保历史记录加载时序正确

## v0.3.0

### 新增功能

#### HTTP 正向代理

- 设置页面新增 HTTP 代理配置区域（代理地址 + 启用开关）
- 启用后，请求调试面板发出的所有 HTTP 请求（普通请求和流式请求）通过指定代理转发
- 支持 Clash、V2Ray 等本地代理工具（如 `http://127.0.0.1:7897`）
- 代理配置自动持久化存储

#### 复制为 curl

- 请求栏新增复制 curl 按钮，一键生成 curl 命令并复制到剪贴板
- 包含 method、URL、headers、body（JSON/Form/Raw）、代理参数（`-x`）

### 技术变更

- `http_client.rs`：`HttpRequest` 新增可选 `proxy` 字段，根据代理配置动态构建 reqwest Client
- `settings.ts`：新增 `ProxyConfig`（`enabled` + `target`）及 `activeProxy` 计算属性

## v0.2.0

### 新增功能

#### Mock 规则 JavaScript 函数模式

- 支持 JavaScript 函数模式，可根据请求数据动态返回响应
- 函数接收 `{ query, body, path }` 参数，分别对应查询参数、请求体和路径参数
- 支持 `formData` 类型请求体解析
- 自动提取路径参数（如 `/api/users/:id` 中的 `id`）

#### 代码格式化与验证

- 使用 `js-beautify` 替换原有格式化功能，支持 JSON 和 JavaScript 代码格式化
- 使用 `acorn` 对 JavaScript 代码进行实时语法校验，显示错误行号和详细信息
- 格式化按钮在 JSON 和 JavaScript 模式下均可使用

#### AI 助手改进

- 改进错误处理，针对 401（API Key 无效）和 402（余额不足）显示友好错误提示
- 修复 CSP 配置，允许 AI 助手访问外部 HTTPS API

### 优化改进

- 添加 `pnpm dev` 命令，启动开发模式时忽略窗口关闭的退出码错误
- Mock 服务器支持 `application/x-www-form-urlencoded` 请求体解析
- JavaScript 模式下跳过 JSON 格式验证，避免误报错误

### 技术栈更新

- 新增依赖：`acorn@^8.16.0`、`js-beautify@^1.15.4`、`@types/js-beautify@^1.14.3`

## v0.1.0

### api-mocker 基于 `Tauri + Vue 3` 的本地 API Mock 与请求调试工具，面向前端联调、接口演示和本地自测场景。

#### Mock 服务器

- 本地 HTTP Mock 服务器，基于 Hyper 实现
- 支持自定义端口配置（默认 3000，范围 1024-65535）
- 服务器状态实时显示（运行中/已停止）
- 应用启动时自动启动 Mock 服务
- 支持路径匹配规则：
  - 精确匹配：`/api/users`
  - 通配符匹配：`/api/*`
  - 路径参数：`/api/users/:id`

#### 接口管理

- 分组管理：支持创建、编辑、删除分组
- 嵌套分组：支持多级分组结构
- 接口 CRUD：创建、编辑、删除接口
- 拖拽排序：支持分组和接口的拖拽排序
- 拖拽移动：支持将接口拖拽到其他分组
- 接口搜索：支持按名称、路径、方法搜索
- URL 唯一性校验：同一方法+路径组合不可重复

#### 请求调试

- HTTP 方法支持：GET、POST、PUT、DELETE、PATCH
- URL 输入：支持相对路径和完整 URL
- Params 配置：可视化添加 URL 查询参数
- Headers 配置：可视化添加请求头
- Body 类型支持：
  - None：无请求体
  - JSON：JSON 格式数据，支持格式化
  - Form：表单数据
  - Raw：原始文本
- JSON 实时校验：编辑时实时检测格式错误
- 自动保存：请求配置自动保存到接口

#### 响应查看

- 状态码显示：HTTP 响应状态码
- 响应时间：请求耗时统计（毫秒）
- 响应头查看：服务器返回的 Headers
- 响应体展示：
  - JSON 格式化显示
  - 图片预览（PNG、JPG、GIF、WebP 等）
  - 二进制文件处理（Base64 编码）
- 流式响应支持：大文件渐进式加载

#### Mock 规则

- 多规则管理：每个接口支持多个 Mock 规则
- 规则切换：一键切换激活的规则
- 规则参数配置：
  - 规则名称
  - HTTP 状态码（200、400、500 等）
  - 响应延迟（毫秒）
  - 响应体（JSON 格式）
- Monaco 编辑器：专业的代码编辑体验
- 编辑器展开：支持全屏编辑模式
- JSON 格式化：一键格式化响应体

#### Mock.js 支持

- 完整 Mock.js 语法支持

#### 环境变量

- 多环境管理：dev、test、prod 预置环境
- 自定义环境：支持添加新环境
- 变量配置：键值对形式配置变量
- 变量启用/禁用：单独控制每个变量
- 变量引用：`{{变量名}}` 语法
- 自动替换：发送请求时自动替换变量
- baseUrl 支持：相对路径自动拼接 baseUrl

#### 历史记录

- 自动保存：每次请求自动记录
- 历史信息：方法、URL、时间、状态码
- 详情查看：完整的请求/响应详情
- 快速恢复：点击历史记录恢复请求配置

#### 数据导入导出

- JSON 格式导出：导出所有分组、接口、规则
- 文件选择器：系统原生文件对话框
- 数据导入：从 JSON 文件恢复数据
- 数据校验：导入时验证数据格式

#### 界面设计

- 三栏布局：图标栏 + 侧边栏 + 主工作区
- 侧边栏宽度可调：拖拽调整宽度（200-400px）
- 深色/浅色主题：支持主题切换
- 响应式设计：适配不同窗口尺寸
- Tab 切换：请求调试、Mock 规则、环境变量、历史记录

#### AI 助手

- AI 悬浮按钮：Mock 规则编辑时显示
- AI 对话框：与 AI 交互生成 Mock 数据
